WFC - это фреймворк для разработки высконагруженных JSON-RPC сервисов (демонов) под различные профили нагрузки. Эта документация по разработке пакетов модулей на WFC. Как подключить разработанный пакет к проекту выполняемого демона описано здесь wfcroot или изучите Демо-проект.

Сборка и зависимости

Зависимости:

  • g++ 4.9 и выше или clang++
  • boost-1.73 и выше
  • cmake
  • git
  • valgrind (опционально)
  • cppcheck (опционально)

Установить свежий boost:

wget https://dl.bintray.com/boostorg/release/1.63.0/source/boost_1_73_0.tar.gz
tar -xvf boost_1_73_0.tar.gz
cd boost_1_73_0
./bootstrap.sh --prefix=/usr/local/boost-1.73
sudo ./b2 install --prefix=/usr/local/boost-1.73

Пропишите в систему BOOST_ROOT или устанавливайте перед запуском, например:

BOOST_ROOT=/usr/local/boost-1.73 cmake ..

WFC подключается к проекту как субмодуль, но можно его собрать независимо, чтобы убедиться что нет проблем со сборкой:

git clone https://github.com/mambaru/wfc.git
cd wfc
make tests

При сборке автоматически инициализируются и загружаются следующие субмодули wfc:

  • cmake-ci - вспомогательные скрипты для cmake и CI
  • faslib - метабиблиотека для работы со списками типов и АОП
  • wjson - очень быстрый декларативный JSON-сериализатор на шаблонах
  • wlog - логер с большими возможностями по кастомизации
  • wflow - очереди и потоки
  • iow - сетевая библиотека
  • wjrpc - библиотека поддержки JSON-RPC
  • wrtstat - для сбора статистики

Разработка демонов, пакетов и библиотек на WFC

Можно выделить три вида проектов:

  • Библиотеки ( Пример: wlog )
  • WFC-пакеты ( Пример: wfc_demo )
  • Демоны ( Пример: demod )

Разработка приложений (демонов) на базе WFC

wfcroot

Разработка WFC-пакетов

Для того чтобы легче визуально отличать пакеты от библиотек, пакетам дают префикс wfc_, например wfc_demo. Для облегчения подключения wfc и написания cmake скриптов будем использовать cmake-ci. Изначально это была папка со скриптами в WFC проекте, но потом эволюционировала в отдельный, независимый проект.
