Подробное описание

Используется для регистрации в глобальном реестре объектов и предоставляет методы для обмена данными в сыром виде.

Методы

◆ reg_io()

void wfc::iinterface::reg_io ( io_id_t  io_id,
std::weak_ptr< iinterface >  itf 
)
virtual
Аргументы
io_idИдентификатор объекта, который вызвал этот метод
itfУказатель на объект с идентификатором io_id

Любой объект системы может уведомить этот объект, что получил на него ссылку, а так-же передать указатель на себя. Например, когда клиент подключается к системе, создается объект TCP-соединения, который может уведомить прикладной объект, о том, что клиент подключился, и передать указатель на себя, через который прикладной объект может передать данные клиенту, не дожидаясь запроса от него.

◆ unreg_io()

void wfc::iinterface::unreg_io ( io_id_t  io_id)
virtual
Аргументы
io_idИдентификатор объекта

Может вызываться, когда объект io_id перестает использовать этот объект.

Переопределяется в wfc::domain_object< ijsonrpc, Config, StatConfig > и wfc::domain_object< ijsonrpc, service_options< ::wjrpc::engine< ::wjrpc::handler< Impl< MethodList > > >::options_type >, statistics_options >.

◆ perform_io()

void wfc::iinterface::perform_io ( data_ptr  d,
io_id_t  io_id,
output_handler_t  handler 
)
virtual
Аргументы
dВходящий буфер
io_idИдентификатор объекта, который вызвал этот метод
handlerОбработчик результата запроса

Обрабатывает запрос d и отправляет ответ через handler (синхронно или асинхронно). Идентификатор io_id может использоваться для определения актуальности handler при асинхронной обработке. Если после вызова perform_io перед вызовом handler произошло unreg_io, то handler можно не вызывать, например выкинуть handler из очереди на обработку. В большинстве случаев, для реализации perform_io не обязательно реализовывать reg_io/unreg_io
